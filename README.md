# BMC
============================================================
世界觀日（Worldview Day）— 合理產出規格 v1.0

關鍵字:世界觀日的目的
內容:

維護「大頭層」的架構清晰度

把卷一、卷二、卷三的架構整理成你看得懂的版本

不追求產出量，而追求：思考乾淨、系統穩固、方向正確

世界觀日 = 你人生 OS 的「系統更新日」

世界觀日的基本責任（做這些即可，不要超過）

關鍵字:三大任務
內容:

整理一份 ASCII 架構圖／總綱片段（卷一 or 卷四 or GitHub）

重讀一小段卷一或卷二或卷三（任選）並補一句心得

確認六主線是否仍然平衡（不需要修改，只檢查）

→ 做到這三點，就已經是“滿分的世界觀日”。

世界觀日的最小完成標準（MVP）

關鍵字:MVP（3–10 分鐘即可）
內容:

產生一份 ASCII「小整理」（20～80 行）

主題永遠是「架構」不是「技術細節」

完成後貼入 GitHub（卷四 or lifeos repo）

例：

BMC 五層架構

DTS → Driver → Register 路徑

六主線一頁式檢查

卷一某章節的小摘要

→ 你剛剛說「把總綱用 ASCII 放到卷四技能篇與 GitHub」——
這件事本身就是世界觀日最標準、最理想的產出。

世界觀日 × 技能日 的差異（最重要）

關鍵字:技能日
內容:

寫 code

畫流程圖

debug / select / SMBus / DTS

用手去做

關鍵字:世界觀日
內容:

不寫 code（或只寫 0～5 行示範）

不畫複雜流程圖（最多 ASCII）

不做實作

用大腦整理結構

→ 技能日 = 小手
→ 世界觀日 = 大頭
兩者不能混。

世界觀日可以產出的三種「成果格式」

關鍵字:格式1：ASCII 架構圖
內容:

20～80 行

放 GitHub / 卷四

例：

關鍵字:格式3：卷一／卷二／卷三 的一句心得
內容:
只需一句，例如：

今日心得：勢的本質不是加速，而是減少阻力讓事情自然往前。


→ 一句話就是「世界觀更新」。

世界觀日不該做的事（避免超載）

關鍵字:避免事項
內容:

不需要重構卷一

不需要寫完整技術文（那是技能日）

不需要整理 500 行 ASCII

不需要完成整段孫子兵法學習

不需要同時做學習（那是心智日）

不需要寫完整 C code

世界觀日只負責看整體，不負責執行。

世界觀日 = 每週的「系統健康檢查」

關鍵字:檢查清單
內容:

本週六主線是否平衡？

技能主線（卷四）方向是否仍正確？

年度目標（卷二）是否有偏移？

心智（卷三）是否過勞？

→ 只需「看」「想」「更新 1 小段 ASCII」。
→ 不需要行動，不需要推進，不需要成果很大。

世界觀日 × 孫子兵法 × 參同契 心法

關鍵字:兵法
內容:
「布陣而後戰。」
→ 世界觀日就是布陣，不是作戰。

關鍵字:參同契
內容:
「一陰一陽之謂道。」
→ 技能日 = 陽（動）
→ 世界觀日 = 陰（靜）
陰陽交替才能長期推進。

                     │        世界觀日（主線）      │
                     └──────────────────────────┘

                           ▼      │
┌──────────────────────────────────────────────────────────┐
│                     世界觀日：定義「Where」                │
│----------------------------------------------------------│
│ - 決定你這個月主題文的大方向（例：GPIO、DTS、SMBus）       │
│ - 建立高階架構：SoC → DTS → Driver → /dev → App           │
│ - 讀 datasheet、框架、流程、Kernel 架構                   │
│ - 產出：架構圖、概念圖、資料流、原因解釋（WHY）            │
└──────────────────────────────────────────────────────────┘

                           │
                           │ 將大方向拆成可實作的小技能
                           ▼

┌──────────────────────────────────────────────────────────┐

│----------------------------------------------------------│
│ - 小坑練習（GPIO、LED、select、debounce、probe log 等）   │
│ - 手寫 C 程式碼、驗證、debug、印 log                     │
│ - 驗證世界觀日的觀念是否真實可跑通                       │
│ - 產出：Code、流程圖、dmesg、i2cdump、測試紀錄           │
│                                                            │
│ ⭐ 技能日提供「證據」：                                     │
│   你真的做過、測過、跑過、debug 過 → 面試最看中這個        │
└──────────────────────────────────────────────────────────┘

                           │
                           │ 整合
                           ▼

┌──────────────────────────────────────────────────────────┐
│                月技術文（主題文）：打造「Proof」            │
│----------------------------------------------------------│
│ - 世界觀日給框架、方向、架構圖（WHY）                     │
│ - 技能日給 C 程式、實測結果、debug（HOW）                 │
│                                                            │
│ → 主題文 = WHY + HOW = 一篇能放 LinkedIn / 面試的作品      │
│                                                            │

│   2. DTS / driver / GPIO / SMBus 知識解釋（世界觀）        │
│   3. 你的程式碼與實測（技能）                               │
│   4. 流程圖、dmesg、驗證（技能）                            │
